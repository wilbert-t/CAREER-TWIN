# Lessons Learned – Career Twin

## Session 1 – 2026-04-07

### Shell state does not persist between Bash tool calls
Each Bash call is a new shell. `source .venv/bin/activate` has no effect on the next call.
**Fix:** Always use full venv binary paths: `.venv/bin/pytest`, `.venv/bin/pip`, `.venv/bin/uvicorn`, `.venv/bin/python`. Put this as a bolded note at the top of every subagent prompt that touches Python.

### Pre-creating directories breaks create-next-app
Running `mkdir -p frontend/{app,components,...}` before `create-next-app` causes it to fail with "directory contains conflicting files".
**Fix:** Let `create-next-app` scaffold the frontend directory itself. Only pre-create backend and knowledge_base dirs.

### python-multipart must be in requirements.txt from day one
FastAPI `UploadFile` requires `python-multipart` installed or file upload endpoints silently fail.
**Fix:** Add `python-multipart` to requirements.txt in Phase 1 setup.

### Always implement mock fallback for LLM services
CV structurer calls Groq API. Without GROQ_API_KEY the route would 500 and block all frontend dev.
**Fix:** Implement `_mock_structure()` that returns plausible demo data when API key is empty. Apply same pattern to every LLM service (`suggest_roles`, `analyze_role_fit`, etc.).

### Tailwind v4 + shadcn/ui CLI may fail — have a manual fallback ready
`npx shadcn@latest init` can fail due to network errors or Tailwind v4 incompatibility.
**Fix:** If CLI fails, manually create `components.json`, add CSS variables to `globals.css`, and copy component files directly. Don't block on the CLI.

## Session 2 – 2026-04-07

### "Load failed" on Safari = fetch() network error, not an HTTP error
Safari reports `TypeError: Load failed` when `fetch()` fails at the network level (server not running, CORS drop, connection reset). This is NOT the same as an HTTP 4xx/5xx — it won't be caught by `if (!res.ok)`.
**Fix:** Wrap `fetch()` itself in try/catch and show a human-readable message ("Cannot connect to server…"). On the backend, always return a proper HTTP response rather than dropping the connection — wrap service calls in try/except so unhandled exceptions don't silently close the socket.

### LLMs often wrap JSON in markdown fences — strip before json.loads
Groq and other LLMs sometimes return ` ```json\n{...}\n``` ` even when instructed not to. `json.loads()` will throw `JSONDecodeError` on fenced content, causing a 500 that drops the connection.
**Fix:** Always strip fences before parsing: `if content.startswith("```"): content = content.split("\n",1)[-1]; content = content.rsplit("```",1)[0].strip()`. Apply this pattern to every LLM service.

### test_upload.py in backend root gets picked up by pytest
A loose `test_upload.py` in `career-twin/backend/` (not in the `tests/` subdirectory) is discovered by pytest and can cause confusion or failures.
**Fix:** Name manual test scripts `manual_*.py` or put them outside the pytest discovery path.

### Subagent worktrees vs main branch
When running subagents with `isolation: "worktree"`, their commits land on a separate branch and must be merged back. When running without isolation, commits go directly to the active branch. For sequential tasks that build on each other, run without isolation to avoid merge overhead.

### Preview_match_score needs Pydantic range validation
A plain `int` field with a comment `# 0–100` is not enforced. Pydantic will accept 150 or -5.
**Fix:** Use `Field(..., ge=0, le=100)` whenever a numeric field has a bounded range. Catch this at model definition time, not in the service layer.
