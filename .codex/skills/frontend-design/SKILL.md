# Frontend Design Skill

Build distinctive, production-grade frontend interfaces with strong visual quality. Avoid generic AI aesthetics. Implement real working code with thoughtful design choices, clean hierarchy, and polished motion.

## When to use this skill
Use this skill when the user asks to:
- build a landing page
- refine a dashboard UI
- create or improve a loading animation
- redesign a page without rebuilding the whole app
- improve spacing, hierarchy, typography, colors, and visual polish
- make a hackathon MVP look presentation-ready
- create frontend code for React, Next.js, or modern web apps

## Core behavior
When handling frontend tasks, do not produce bland template-like UI.

Always:
- choose a clear aesthetic direction
- make deliberate typography choices
- use a controlled, coherent color system
- create polished spacing and layout rhythm
- make motion feel smooth, not noisy
- ensure the output feels like a real product, not an AI mockup

## Design thinking before coding
Before writing code, infer and commit to a clear direction:

### 1. Purpose
Identify:
- what this interface does
- who uses it
- what the primary action is
- what the first impression should communicate

### 2. Aesthetic direction
Choose one strong direction and stay consistent. Examples:
- minimal and premium
- editorial and modern
- soft and calm
- bold and high-contrast
- refined corporate
- futuristic but restrained
- playful but structured

Do not mix too many styles at once.

### 3. Constraints
Respect:
- framework already in use
- existing code structure
- performance needs
- accessibility basics
- user’s current branding and page flow

## Non-negotiable principles

### Avoid generic AI aesthetics
Do not produce:
- random gradients everywhere
- overused glassmorphism
- flashy but meaningless animations
- cluttered dashboards
- oversized floating cards with no hierarchy
- visually busy hero sections
- stock “AI platform” layouts with no personality

### Preserve existing structure when possible
If the project already exists:
- do not rebuild from scratch unless necessary
- refine and upgrade the current structure
- improve hierarchy, layout, and polish first
- keep changes realistic and maintainable

### Think in systems
Define and apply a consistent system for:
- headings
- body text
- spacing
- radius
- border treatment
- shadow usage
- button styles
- icon styling
- accent color usage

## Typography guidance
Typography should create hierarchy immediately.

Prefer:
- strong, clean headings
- softer supporting text
- clear contrast between heading, body, and caption
- consistent line height and spacing

Avoid:
- weak headings
- body text that is too large
- too many font sizes
- inconsistent weights across similar UI elements

## Color guidance
Choose a controlled palette with roles:
- background
- primary text
- secondary text
- accent
- surface/card
- border/divider
- success/warning/error if needed

Use accent colors intentionally, not everywhere.

For products like Career Twin or career tools, default toward:
- calm neutrals
- trustworthy blues
- warm off-white or beige backgrounds when appropriate
- restrained contrast with clear CTA emphasis

## Layout guidance
Prioritize:
- strong hierarchy
- generous spacing
- aligned content blocks
- clean grouping
- obvious primary action

Good frontend layout should feel:
- organized
- breathable
- easy to scan
- visually balanced

## Motion guidance
Animations should feel smooth and intentional.

Prefer:
- opacity transitions
- transform-based motion
- soft easing
- coordinated timing
- subtle blur or scale only when it supports the experience

Avoid:
- choppy transitions
- sudden jumps
- too many moving layers
- overly long intros
- animation that delays usability

## Loading / intro transition guidance
When building a loading screen or intro transition:
- make it feel like part of the product
- ensure the ending blends naturally into the landing page
- keep branding consistent between intro and final page
- prioritize smoothness over complexity
- use elegant, controlled motion

For transition work, evaluate:
- background continuity
- color continuity
- pacing
- animation overlap
- whether the destination page reveal feels natural

## Landing page guidance
For landing pages:
- make the value proposition clear above the fold
- keep CTA obvious
- reduce clutter
- make demo screenshots look clean
- show the product’s purpose immediately

The first section should answer:
- what is this
- who is it for
- why should I care
- what do I do next

## Dashboard guidance
For dashboards:
- reduce visual noise
- make cards scannable
- keep sidebars simple
- use accent colors sparingly
- ensure charts and metrics do not compete equally for attention
- make the page look calm and usable

## React / Next.js implementation guidance
When working in React or Next.js:
- preserve existing component structure where possible
- keep components readable
- avoid unnecessary rewrites
- follow the project’s current styling approach
- if Tailwind is already used, stay in Tailwind
- if CSS modules or styled-components are already used, stay consistent

Use animation libraries only when justified. If simple CSS transitions can achieve the effect cleanly, prefer that.

## Hackathon mode
When the project is a hackathon MVP:
- optimize for fast visual impact
- prioritize first impression
- choose high-value, low-risk UI improvements
- make the interface screenshot-friendly
- aim for demo polish, not unnecessary complexity

## Output style
When responding to frontend tasks:
1. infer the page’s purpose
2. identify the visual problems
3. choose a strong direction
4. make targeted, production-ready code changes
5. keep the result distinctive, polished, and coherent

When explaining changes, focus on:
- hierarchy
- consistency
- motion
- readability
- design rationale

Do not over-explain theory unless the user asks.

## Special instruction for this project style
For career-tech / portfolio / CV-analysis products:
- make the experience feel trustworthy and polished
- avoid overly playful visuals unless requested
- emphasize clarity, guidance, and professionalism
- use clean onboarding flow
- make upload and analysis steps feel approachable
- ensure the UI looks good in a presentation deck
